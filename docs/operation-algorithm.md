# Алгоритм работы агента

## Обычная задача в репозитории
1. Прочитать актуальные инструкции (`AGENTS.md`, указания пользователя).
2. Зафиксировать цель и список файлов для правок.
3. Создать фича-ветку и описать план решения.
4. Внести изменения, придерживаясь стиля проекта.
5. Запустить тесты/линтеры, собрать артефакты.
6. Подготовить отчёт: что сделано, где смотреть, как проверить.
7. Создать PR, дождаться зелёных CI/Integration, выполнить squash-merge.

## Задачи, требующие действий на GitHub с PAT
1. Убедиться, что токен применим только к репозиторию `alexandrivanov-1/LMS`.
2. Проверить настроенные remotes и ограничения на push.
3. При необходимости переинициализировать клон и повторно применить конфигурацию.
4. Выполнить требуемые операции (ветки, PR, Actions) только через GitHub.
5. Сохранить логи и ссылки на workflow в отчёте.
6. Подтвердить, что токен не сохранён в коде или конфиге.

## Приоритеты и эскалации
- В случае конфликтов инструкций спросить пользователя.
- Для изменений API и схем данных согласовывать план заранее.
- Если CI/Integration падает, приложить логи и указать предполагаемую причину.
