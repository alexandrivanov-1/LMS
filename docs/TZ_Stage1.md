# Stage 1 Technical Specification — Исправленная версия (курс 44-ФЗ/223-ФЗ)

## 1. Project Overview
Stage 1 создаёт фундамент знаний для курса по закупкам (44-ФЗ/223-ФЗ): загрузка источников, нормализация, векторное и графовое представления, «маска знаний» (атомы+контексты), поиск с цитатами «на дату», базовые активности (банк вопросов, карточки, аудио-брифинг). Публичный демо-UI без авторизации; админ-UI закрыт.

## 2. Objectives
- Ingest: профстандарты 625н/626н, 44-ФЗ/223-ФЗ, методички, презентации, лекции, тесты (XLSX).
- Разметка единиц нормы до подпункта с `valid_from/valid_to` и стабильными ID.
- Индексы: Qdrant (вектор) + Neo4j (граф); гибридный поиск с цитатами и параметром `as_of`.
- «Маска знаний»: атомы (тип, Блум, пререквизиты, цитаты) и контекст-профили.
- Активности: импорт MCQ из Excel, шаблонная генерация MCQ, карточки (CSV/Anki), аудио-брифинг (локальный TTS).
- Публичный UI: «Карта курса» (упрощённый граф) + демо-поиск с цитатами.

## 3. Success Metrics
- Индексация ≥ 200 документов; корректная разметка норм до подпункта.
- ≥ 100 атомов знаний со ссылками на нормы.
- Поиск p95 ≤ 800 мс; все ответы с корректными цитатами.
- Импортирован банк ≥ 300 MCQ; сгенерирован MP3-аудио-брифинг.
 
## 4. Scope
### 4.1 In Scope
- Загрузка/очередь, парсинг (PDF/DOCX/PPTX/XLSX/URL), OCR.
- Хранилища: Postgres, MinIO, Qdrant, Neo4j.
- Единицы нормы, чанки, цитаты; гибридный поиск (BM25+вектор+rerank) `as_of`.
- Маска знаний: CRUD атомов, контекст-профили, граф-виз (админ).
- Публичный UI без логина: карта курса и демо-поиск.
- MCQ импорт + генерация шаблонов; карточки; аудио-брифинг (Piper).
- REST-API Stage 1 (ingest/atoms/search/graph/questions/cards).

### 4.2 Out of Scope
- Авторизация/SSO/RBAC, многоарендность.
- Задания/оценки/журналы, дискуссии/анонсы, LTI.
- Облачные CDN/мульти-регион.
- Авто-Delta-Engine и Patch-Builder (Stage 2–3).

## 5. Stakeholders
Product Owner, Engineering Lead, Design Lead, QA Lead, Методист (закупки).

## 6. Roles
- Admin/Editor (закрыт по IP): импорт, разметка, маска, артефакты.
- Public Visitor (без логина): карта курса, демо-поиск, глоссарий.

## 7. Functional Requirements
1) Ingest & Normalize: реестр источников, реквизиты/лицензии, `valid_from/valid_to`, разметка до подпункта, чанки (страница/абзац/слайд/таймкод), dedup.  
2) Index & Search: Qdrant (вектор 768), фильтры по нормам и `as_of`, обязательные цитаты (норма+координаты).  
3) Mask: атомы (тип, Блум, гранулярность, пререквизиты, заблуждения, цитаты, статус, версия), контекст-профили; связи в Neo4j.  
4) Activities: MCQ импорт (4 варианта/1 верный, explanation, привязка, `valid_from/to`), генерация черновиков; карточки; аудио-брифинг (Piper).  
5) UI: админ (загрузка, статусы, редактор атомов, граф-виз), публичная карта курса и поиск.

## 8. Non-Functional
- On-prem: Ubuntu 22.04, Docker Compose, Nginx, TLS.
- Перф: p95 поиска ≤ 800 мс (до 200k чанков).
- Надёжность: Celery-очереди, health-checks, авто-рестарт.
- Бэкапы: Postgres, MinIO, Neo4j, Qdrant (cron + тест восстановления).
- Безопасность: админ — IP-allowlist/Basic-Auth; публичный поиск — rate-limit/капча.
- Локализация: RU (i18n-скелет).

## 9. Milestones
M1: Infra → M2: Ingest/Parser/Indexer → M3: Mask → M4: Search+Public UI → M5: Activities.

## 10. Risks & Mitigations
Качество парсинга PDF; юридическая точность (строгие цитаты); производительность индексации (очереди, батчи).

## 11. Acceptance Criteria
- Импорт/индексация стартового корпуса; ≥ 100 атомов с цитатами и связями.
- Поиск `as_of` с корректными цитатами.
- Импорт ≥ 300 MCQ; экспорт карточек CSV/Anki; MP3-брифинг сгенерирован.
- Публичная карта курса рендерится < 2 с; админ-операции без ошибок.
