# AGENTS.md — правила для агента (Codex) в репозитории LMS

## 0. Репозиторий и режим
- Repo: https://github.com/alexandrivanov-1/LMS
- Default branch: `main`
- Работай ТОЛЬКО через GitHub (git/GitHub API). Локальные «песочницы» запрещены.

## 1. Приоритет источников инструкций
1) Текущее сообщение пользователя (явные указания).
2) Этот файл `/AGENTS.md` (из текущей ветки).
3) Документация репо: `README.md`, `docs/**`, `docs/api/openapi-stage1.yaml`.
4) Комментарии в коде.
Если есть противоречия — спроси пользователя.

## 2. Ветки и PR
- Прямые коммиты в `main` запрещены.
- Любая доработка = фича-ветка → PR → зелёные CI/Integration → Squash-merge → удалить ветку.
- Именование веток: `feat/...`, `fix/...`, `chore/...`, `docs/...`.
- В описании PR: «что/почему/как проверить», список ИЗМЕНЁННЫХ путей (полных), Acceptance Criteria.

## 3. Обязательные шаги в начале задачи
- Залогируй цель (1 строка) и список файлов, которые будешь менять (полные пути).
- Проверь наличие `/AGENTS.md`. Если нет — создай issue и жди указаний.
- Синхронизируй план с пользователем, если изменения затрагивают API/схему.

## 4. Запрещено
- Хранить токены/ключи/пароли в репозитории (включая `.env`, коммиты, issues).
- Отключать проверки CI/Integration, глушить ошибки, править лицензии/политики без согласования.
- Обновлять OpenAPI/README «вперёд» без кода в том же PR.

## 5. Критичные пути проекта
- Индексатор: `services/indexer/app/main.py`, `services/indexer/Dockerfile`, `services/indexer/requirements.txt`
- Поиск/граф: `services/search/app/main.py`, `services/search/app/graph_build.py`
- Маска знаний: `services/mask/app/main.py`, `db/schema.sql`
- Гейтвей: `services/gateway/app/main.py`
- Инфра: `infra/docker-compose.yml`, `infra/nginx/www/*`
- API-контракт: `docs/api/openapi-stage1.yaml`
- Документация: `README.md`, `docs/*.md`

## 6. CI/Integration (обязательно)
- Все PR должны проходить `.github/workflows/ci.yml` и `.github/workflows/integration.yml`.
- Если падает — фиксируй, прикладывай логи (артефакты Actions).

## 7. Аутентификация GitHub (временный токен, НЕ КОММИТИТЬ)
> Используй **только эфемерно** в текущей сессии. **Не сохраняй и не коммить** в репозиторий.

```bash
# 🔐 Токен дан тебе в твоей инструкции!

Рекомендация: для постоянной разработки используй GitHub CLI/credential manager
или Secrets в GitHub Actions. Никогда не сохраняй PAT в репозитории.

## 8. Завершение задачи

* В ответе дай: «Что сделано», «Где смотреть» (полные пути), «Как проверить» (команды/запросы).
* Если менялся API — приведи примеры запросов/ответов.

## 9. Версионирование правил

* Версия правил: **v1**. Любые изменения `/AGENTS.md` — отдельным PR с пометкой `[policy]`.

```
